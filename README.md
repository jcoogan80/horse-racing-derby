# Derby Value — Web App

Mobile-first horse racing analytics for Derby day. Three tabs:

- **Dashboard** — live overlay signals loaded from `dashboard_data.json`
- **Hedge Calc** — in-browser Pick 4/5/6 last-leg hedge calculator
- **Profiles** — static KEE/GP baseline overlay profiles (April 2026)

Everything runs in the browser. No backend, no frameworks, no CDN
dependencies. Works offline after first load via service worker.

---

## File Reference

| File | What it does |
|---|---|
| `index.html` | Entire app — HTML, CSS, and JavaScript in one file |
| `sw.js` | Service worker — caches the app for offline use |
| `dashboard_data.json` | Generated by the export script; pushed to GitHub after each scrape |
| `netlify.toml` | Netlify config — sets publish dir and cache headers |
| `.github/workflows/deploy.yml` | Auto-deploys to Netlify on every push to `main` |

The Python files (`HorseRacing.py`, `derby_value.py`, `hedge_calculator.py`,
`horse_racing.db`) live on your local machine and are **never** pushed here.

## First-Time Setup

### 1. Connect to Netlify

Log in at [netlify.com](https://netlify.com) and create a new site:

- **New site** → **Import an existing project** → connect your GitHub account
- Select the `horse-racing-derby` repository
- Build command: *(leave blank)*
- Publish directory: `.`
- Click **Deploy**

### 2. Add GitHub Secrets for Auto-Deploy

The GitHub Action needs two secrets so it can deploy on every push.

**Get your Netlify tokens:**

1. Netlify Auth Token:
   - Go to **User Settings → Applications → Personal access tokens**
   - Click **New access token**, name it `github-actions`, copy the value

2. Netlify Site ID:
   - Go to your site in Netlify → **Site configuration → General**
   - Copy the **Site ID** (looks like `a1b2c3d4-...`)

**Add them as GitHub secrets:**

1. In your GitHub repo, go to **Settings → Secrets and variables → Actions**
2. Click **New repository secret** and add:
   - Name: `NETLIFY_AUTH_TOKEN` — paste the personal access token
   - Name: `NETLIFY_SITE_ID` — paste the site ID

After this, every push to `main` auto-deploys.

## Updating Dashboard Data (After Each Scrape)

The `export_dashboard_data.py` script lives in your local project folder
(not in this repo). Run it from there after scraping new race data:

```
cd "C:\Users\jason\Desktop\HorseRacing Project"
python export_dashboard_data.py
```

This reads `horse_racing.db` and writes `web/dashboard_data.json`.

To update only specific tracks:
```
python export_dashboard_data.py CD
python export_dashboard_data.py CD KEE GP
```

### Push the Updated JSON to GitHub

After running the export script, push the new `dashboard_data.json`:

```
cd "C:\Users\jason\Desktop\HorseRacing Project\web"
git add dashboard_data.json
git commit -m "update dashboard data"
git push
```

GitHub Actions picks this up and deploys to Netlify automatically.
The live site updates in about 30 seconds.

## Derby Day Workflow

**Before the card starts:**
```
python export_dashboard_data.py
cd web
git add dashboard_data.json && git commit -m "pre-derby baseline" && git push
```
Open the live site on your phone. Check the Dashboard tab for any
existing CD overlays.

**After each race posts (5-10 min after official result):**
1. Scrape via `HorseRacingGUI.py` — select CD, pick today's date, run
2. `python export_dashboard_data.py`
3. `git add dashboard_data.json && git commit -m "race N update" && git push`
4. Refresh the Dashboard tab on your phone

**When a ticket is alive going into the last leg:**
1. Switch to the **Hedge Calc** tab on your phone
2. Enter ticket cost and estimated payout from the tote board
3. Enter field size, tap Generate
4. Check your horses, enter current odds for all horses
5. Enter your hedge budget (what you have in your pocket)
6. Tap CALCULATE HEDGES
7. Scenario Table shows every horse sorted by win probability
8. Budget section shows optimal allocation if budget can't cover all

## How the Offline Cache Works

The service worker (`sw.js`) caches `index.html` and `sw.js` on first
load. Once cached, the app loads instantly with no cell service.

- `dashboard_data.json` uses **network-first**: always tries to fetch
  fresh data, falls back to the last cached version when offline.
  A yellow banner appears at the top when serving cached data.
- All other files use **cache-first**.

The app is fully functional offline for the hedge calculator and
track profiles tabs. The dashboard tab shows the last data you loaded
while online.
